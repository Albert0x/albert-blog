## 起因

入了一台戴尔 OptiPlex 7020 MT Plus，原厂预装 Ubuntu Server 22.04。
这配置跑单系统太浪费了 —— 用它当家用 **PVE 虚拟化平台**，可以同时跑工业系统的 VM、软路由、NAS、AI 推理服务。

这篇是我自己装一遍的记录，**主要给下次重装的我自己看**。

## 硬件

| 项     | 规格                                    | PVE 评估              |
| ------ | --------------------------------------- | --------------------- |
| CPU    | Intel i7-14700（8P+12E，20 核 28 线程） | 支持 VT-x / VT-d      |
| 内存   | 128GB DDR5                              | 跑十几台 VM 不慌      |
| 系统盘 | 512GB NVMe SSD（三星 PM9C1a）           | 装 PVE + VM 系统盘    |
| 数据盘 | 4TB 机械                                | VM 数据 + 备份        |
| 网卡   | 5 口千兆（板载 1 + 扩展 4）             | 网络隔离 / 软路由直通 |
| 显示器 | 戴尔 E2425HM 23.8 寸（DP）              | 装机用                |

## 准备工作

### 要准备的材料

- U 盘 8GB 以上（**做之前先备份 U 盘里的东西**，会被清空）
- PVE 9.2 ISO（约 1.5GB）
- **Rufus**（Windows 刻录工具）
- 网线 + Windows 电脑（远程访问 PVE Web 界面用）

### 下载 PVE ISO

官方下载页：
https://www.proxmox.com/en/downloads/proxmox-virtual-environment/iso

下载 `proxmox-ve_9.2-x.iso`。

### 做启动 U 盘（关键：用 DD 模式）

1. 插 U 盘，打开 Rufus
2. **设备** → 选 U 盘
3. **引导类型** → 选刚下的 PVE ISO
4. **分区类型** → MBR
5. **目标系统** → BIOS 或 UEFI
6. 点 **开始**
7. 弹出写入模式窗口 → **选「以 DD 镜像模式写入」** → 确定
8. 等 3 分钟左右完成

> ⚠️ **必须选 DD 模式**。ISO 模式刻录的 PVE 经常无法启动，
> 我第一次没注意选了 ISO 模式，反复进不去安装界面，重刻一次才解决。

## BIOS 设置

把 U 盘插到服务器上，开机狂按 `F2` 进 BIOS。

需要改的有 3 项：

### 1. 开启虚拟化

`左侧菜单 → Virtualization Support`

| 选项                                   | 设置   | 说明                  |
| -------------------------------------- | ------ | --------------------- |
| Intel Virtualization Technology (VT-x) | **ON** | CPU 虚拟化，必须开    |
| VT for Direct I/O (VT-d)               | **ON** | PCIe 直通，强烈推荐开 |

![戴尔 BIOS Virtualization Support 页面，VT-x 和 VT-d 都开启](/images/posts/pve/bios-virtualization.jpg)

> VT-d 不开也能装 PVE，但后续要做 GPU 直通、网卡直通就得回来重启再开。
> **一次开好省事**。

### 2. 关闭 Secure Boot

`左侧菜单 → Boot Configuration → Secure Boot → Enable Secure Boot → OFF`

![戴尔 BIOS Security 页面，Secure Boot 关闭](/images/posts/pve/bios-secure-boot.jpg)

> Secure Boot 会拦着 PVE 启动，必须关。

### 3. 启动顺序

`左侧菜单 → Boot Configuration → Boot Sequence`

确认 USB 设备排在第一位（U 盘插入后会自动出现在列表里）。

![戴尔 BIOS Boot Configuration 页面](/images/posts/pve/bios-boot-config.jpg)

改完点 `APPLY CHANGES` 保存 → `EXIT` 退出，服务器自动重启。

## 安装 PVE 9.2

重启会从 U 盘进 PVE 安装菜单。

### Step 1：选安装方式

选 **`Install Proxmox VE (Graphical)`** → 回车 → 等图形界面加载（约 30 秒）→ `I agree` 同意许可。

![PVE 9.2 安装向导欢迎页](/images/posts/pve/pve-welcome.jpg)

### Step 2：选磁盘

- **Target Harddisk** → 选 512GB SSD（`/dev/nvme0n1`）

![PVE 安装向导磁盘选择页 - nvme0n1 三星 PM9C1a 476.94GB](/images/posts/pve/installer-disk.jpg)

- 点 **Options**：

| 选项       | 设置                 |
| ---------- | -------------------- |
| Filesystem | **ext4**（单盘稳妥） |
| hdsize     | 476（全部容量）      |

![PVE 安装向导 Hardisk options 弹窗 - 文件系统选 ext4](/images/posts/pve/installer-fs-options.jpg)

> ZFS 也可以，能用快照 / 压缩，但吃内存（建议至少 1GB/1TB）。

### Step 3：地区时区

| 选项            | 填            |
| --------------- | ------------- |
| Country         | China         |
| Timezone        | Asia/Shanghai |
| Keyboard Layout | U.S. English  |

### Step 4：账户

- **Password**：自己定一个强密码，**记好**（忘了只能通过控制台重置）
- **Email**：写一个能收信的（PVE 系统告警会发到这里）

### Step 5：网络配置（关键）

| 选项                 | 我填的               |
| -------------------- | -------------------- |
| Management Interface | `nic0`（第一块网卡） |
| Hostname (FQDN)      | `pve.local`          |
| IP Address (CIDR)    | `192.168.31.111/24`  |
| Gateway              | `192.168.31.1`       |
| DNS Server           | `192.168.31.1`       |

> ⚠️ **必须设静态 IP**。PVE Web 控制台靠 IP 访问，DHCP 给的 IP 换一次你就找不到了。

### Step 6：装！

Summary 页检查无误 → `Install` → 等 5-10 分钟。

装完提示 `Reboot` 时 **先拔 U 盘** 再点（不然又会从 U 盘进安装界面）。

## 首次访问 Web 界面

服务器重启完，屏幕显示：

```
Welcome to the Proxmox Virtual Environment.
Please use your web browser to configure this server - connect to:
  https://192.168.31.111:8006/
```

![PVE 安装完成后控制台显示 Web 管理界面地址](/images/posts/pve/web-url.jpg)

在同一局域网的电脑浏览器访问 `https://192.168.31.111:8006`：

- 浏览器提示证书不安全 → **高级 → 继续访问**（自签名证书，正常）
- 用户名 `root` / 密码刚才设的 / 领域 `Linux PAM standard authentication`
- 登录后会弹「无有效订阅」提示 → 点 OK 关掉，不影响使用（后面会一行命令永久关掉）

## 系统初始化（SSH 进去配）

Windows 上用 PowerShell 或 Xshell 连：

```bash
ssh root@192.168.31.111
```

### 1. 清掉企业订阅源

PVE 默认带付费企业源，不清掉 `apt update` 会报 401。

```bash
echo "" > /etc/apt/sources.list.d/pve-enterprise.sources
echo "" > /etc/apt/sources.list.d/ceph.sources
# 老版本路径（如果存在）
[ -f /etc/apt/sources.list.d/pve-enterprise.list ] && \
  cat > /etc/apt/sources.list.d/pve-enterprise.list << 'EOF'
EOF
```

### 2. 加免费源

```bash
echo "deb http://download.proxmox.com/debian/pve trixie pve-no-subscription" \
  > /etc/apt/sources.list.d/pve-no-subscription.list
```

> PVE 9.x 基于 Debian 13 (trixie)。如果是 PVE 8.x 把 `trixie` 换成 `bookworm`。

### 3. 先用官方源更新一次

```bash
apt update && apt dist-upgrade -y
```

国外源慢，第一次更新耐心等。

### 4. 换成国内镜像（关键加速）

更新完后换中科大镜像，以后再 `apt update` 速度起飞：

```bash
echo "deb https://mirrors.ustc.edu.cn/proxmox/debian/pve trixie pve-no-subscription" \
  > /etc/apt/sources.list.d/pve-no-subscription.list
apt update
```

### 5. 关闭无订阅弹窗

PVE 每次登录会弹「未购买订阅」窗口，烦。一行命令永久关掉：

```bash
sed -i.bak "s/NotFound/Active/g" /usr/share/perl5/PVE/API2/Subscription.pm
systemctl restart pveproxy
```

刷新 Web，弹窗就没了。
（PVE 升级后这行可能会被覆盖，下次升级再执行一遍。）

## 4TB 机械盘初始化

我打算把 4TB 分两个区：**3.5TB 当 VM 数据盘 + 142GB 当备份盘**。
分两个区的好处是数据和备份物理隔离，备份不会被数据盘满了拖累。

### 1. 看一下盘的设备名

```bash
lsblk
```

4TB 盘一般是 `/dev/sda`。

### 2. 分区（GPT 表）

```bash
fdisk /dev/sda
```

在 `fdisk` 交互界面依次输入：

```
g          # 创建 GPT 分区表
n          # 新建第一个分区
回车        # 分区号默认 1
回车        # 起始扇区默认
+3584G     # 第一个分区 3.5TB
n          # 新建第二个分区
回车回车回车  # 全部默认，用完剩余空间
w          # 写入并退出
```

得到：

| 分区        | 大小     | 用途      |
| ----------- | -------- | --------- |
| `/dev/sda1` | 3.5TB    | VM 数据盘 |
| `/dev/sda2` | 约 142GB | 备份存储  |

### 3. 格式化

```bash
mkfs.ext4 /dev/sda1
mkfs.ext4 /dev/sda2
```

### 4. 挂载（开机自动）

```bash
mkdir -p /mnt/data-hdd /mnt/backup-hdd
echo "/dev/sda1  /mnt/data-hdd    ext4  defaults  0  2" >> /etc/fstab
echo "/dev/sda2  /mnt/backup-hdd  ext4  defaults  0  2" >> /etc/fstab
mount -a
systemctl daemon-reload
df -h | grep hdd
```

### 5. 在 PVE Web 加存储

`数据中心 → 存储 → 添加 → 目录(Directory)`，加两个：

| ID           | 目录              | 内容类型 |
| ------------ | ----------------- | -------- |
| `data-hdd`   | `/mnt/data-hdd`   | 磁盘映像 |
| `backup-hdd` | `/mnt/backup-hdd` | 备份     |


## 最终的存储规划

| 存储名       | 类型     | 大小       | 用途                |
| ------------ | -------- | ---------- | ------------------- |
| `local`      | 目录     | ~100GB SSD | ISO 镜像、CT 模板   |
| `local-lvm`  | LVM-Thin | ~373GB SSD | VM 系统盘（高性能） |
| `data-hdd`   | 目录     | 3.5TB HDD  | VM 数据盘           |
| `backup-hdd` | 目录     | 142GB HDD  | VM 备份             |

## 验证安装

```bash
pveversion          # 看 PVE 版本
free -h             # 确认 128GB 内存识别正常
lsblk               # 确认两块盘都在
ip a                # 看网卡状态
df -h               # 看挂载点
egrep -c '(vmx|svm)' /proc/cpuinfo  # 返回 > 0 表示虚拟化正常
```

## 装过程中遇到 / 听说过的几个坑

| 问题                | 原因               | 解决                                            |
| ------------------- | ------------------ | ----------------------------------------------- |                        |
| `apt update` 报 401 | 企业源没清干净     | 清空 `pve-enterprise.sources` 和 `ceph.sources` |
| Web 界面打不开      | IP / 协议 / 端口错 | 必须是 **https**://192.168.31.111:**8006**      |
| 登录弹订阅警告      | 没买订阅（正常）   | 直接点 OK 或执行第 5 步关闭命令                 |
| U 盘装不进系统      | Rufus 没选 DD 模式 | 重刻，**用 DD 镜像模式写入**                    |
| 装完又进安装界面    | 重启没拔 U 盘      | 重启那一瞬间把 U 盘拔了                         |
